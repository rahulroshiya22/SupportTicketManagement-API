‚ö†Ô∏è Important: Backward Transitions Forbidden

Only forward steps are allowed. Attempting RESOLVED ‚Üí IN_PROGRESS or OPEN ‚Üí RESOLVED returns 400 Bad Request. Each status change is automatically recorded in ticket_status_logs with the user who made the change and timestamp.

Validation
Business Rules

üìè Title Length

Minimum 5 characters. Returns 400 if shorter.

üìù Description Length

Minimum 10 characters. Returns 400 if shorter.

üî¢ Priority Enum

Must be exactly: LOW, MEDIUM, or HIGH. Not free text.

üî¢ Status Enum

Must be exactly: OPEN, IN_PROGRESS, RESOLVED, or CLOSED.

üö´ Assignment Rule

Tickets cannot be assigned to users with role USER. Only MANAGER or SUPPORT can receive assignments. Returns 400 if violated.

üîê Password Hashing

All passwords must be stored as BCrypt hashes. Never store plaintext. Use BCrypt.Net-Next in .NET 8.

üéüÔ∏è JWT Required

Every endpoint except /auth/login requires Authorization: Bearer <token> header.

üîÅ Status Transition

Only one step forward at a time. newStatus = (int)currentStatus + 1 must be true.

HTTP Status Codes Used
Code Meaning When
200 OK Success Successful GET, PATCH
201 Created Resource created Successful POST (create ticket, user, comment)
204 No Content Success, no body Successful DELETE
400 Bad Request Validation error Invalid input, bad status transition, assign to USER

403 Forbidden Role insufficient Token valid but role lacks permission

.NET 8 Implementation
Build Steps

Create Project

dotnet new webapi -n SupportTicketManagement.API

Install NuGet Packages

EF Core, JWT Bearer, Swashbuckle, BCrypt.Net-Next, System.IdentityModel.Tokens.Jwt (see Packages tab)

Create Enums

RoleName { Manager, Support, User } ¬∑ TicketStatus { Open, InProgress, Resolved, Closed } ¬∑ TicketPriority { Low, Medium, High }

Create EF Core Entities

Role, User, Ticket, TicketComment, TicketStatusLog ‚Äî with navigation properties and FK constraints using OnDelete(DeleteBehavior.Cascade) for comments/logs

Configure AppDbContext

Map enums as strings via HasConversion<string>(). Configure both FK relationships on Ticket (created_by/assigned_to both point to User ‚Äî use Restrict to avoid cascade conflicts).

Seed Database

On startup: insert 3 roles if not present. Insert initial MANAGER user with BCrypt-hashed password. Called via DbSeeder.Seed(db) in Program.cs.

Configure JWT in Program.cs

Add AddAuthentication(JwtBearerDefaults.AuthenticationScheme).AddJwtBearer(...). Store JWT Key, Issuer, Audience in appsettings.json.

Create DTOs

LoginDTO, CreateUserDTO, CreateTicketDTO (with MinLength validation), AssignDTO, UpdateStatusDTO, CommentDTO. Use [Required][MinLength][EmailAddress] attributes.

Implement Controllers

AuthController (no auth), UsersController ([Authorize(Roles="Manager")]), TicketsController (mixed roles), CommentsController (custom access check per request)

Status Transition Validation

In TicketService: if((int)newStatus != (int)currentStatus + 1) throw BadRequest. Then INSERT into TicketStatusLog in same transaction.

Configure Swagger with JWT UI

Add AddSecurityDefinition("Bearer",...) in AddSwaggerGen. Access at /swagger in development mode.

Run Migrations & Test

dotnet ef migrations add Init ‚Üí dotnet ef database update ‚Üí Test with Swagger UI or Postman using the seeded admin credentials.
